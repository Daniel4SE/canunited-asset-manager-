## 📋 产品概述

**CANUnited Asset Manager** 是一款革命性的多厂商电气资产智能管理平台，专为现代化电力配电系统设计。平台打破了传统单一厂商管理系统的局限性，实现对 Schneider Electric、ABB、Siemens、Eaton、Bosch Rexroth 等主流电气设备厂商的统一管理和深度集成。

通过先进的物联网(IoT)传感器网络、机器学习算法和实时数据分析，CANUnited 为企业提供前所未有的资产可视化、健康监测和预测性维护能力。

## ✨ 核心功能

### 1. 📊 统一健康仪表盘 (Unified Health Dashboard)

实时监控所有电气资产的运行状态，一目了然地掌握整个配电系统的健康状况。

| 功能 | 描述 |
|------|------|
| 综合健康评分 | 0-100分制，综合评估设备状态 |
| 多维度分析 | 电气/热力/绝缘/机械四维健康指标 |
| 实时告警 | 关键/高/中/低四级告警分类 |
| 趋势分析 | 历史健康趋势可视化 |

### 2. 🏭 多厂商设备管理 (Multi-Vendor Asset Management)

支持业界主流电气设备厂商，提供标准化的设备管理体验。

**支持的厂商：**

| 厂商 | 设备类型 |
|------|----------|
| **Schneider Electric** | 断路器、变压器、配电柜、智能电表 |
| **ABB** | 中压开关柜、VFD变频器、继电保护 |
| **Siemens** | SIPROTEC 保护系统、低压配电 |
| **Eaton** | 低压断路器、UPS电源 |
| **Bosch Rexroth** | 工业控制系统、驱动设备 |
| **通用 IoT** | 第三方传感器、智能设备 |

### 3. 🔮 预测性维护 (Predictive Maintenance)

基于机器学习的智能预测引擎，提前识别潜在故障风险。

- **剩余使用寿命 (RUL)** 预测
- **故障风险评估** 与预警
- **最佳维护窗口** 推荐
- **备件需求** 预测

### 4. 🗺️ 电气拓扑可视化 (Electrical Topology)

直观展示电气系统的物理连接关系和能量流向。

- **单线图** 自动生成
- **关键路径** 分析
- **故障影响域** 可视化
- **负载分布** 热力图

### 5. 📱 QR码快速识别 (QR Code Integration)

每台设备配备唯一QR码，实现快速资产识别和信息访问。

- 扫码即达设备详情页
- 现场维护信息快速查询
- 资产盘点效率提升 **80%**

### 6. 📈 跨厂商分析报表 (Cross-Vendor Analytics)

突破性的跨厂商对比分析能力，为采购和运维决策提供数据支持。

- **厂商可靠性对比**
- **维护成本分析**
- **故障率统计**
- **性能基准测试**

## 📊 健康评分体系

CANUnited 采用独创的 **Unified Health Score (UHS)** 体系，将复杂的设备状态简化为直观的0-100分制评分。

### 评分维度

| 维度 | 权重 | 监测参数 |
|------|------|----------|
| ⚡ 电气健康 | 30% | 电流、电压、功率因数、谐波 |
| 🌡️ 热力健康 | 25% | 温升、热点、散热效率 |
| 🛡️ 绝缘健康 | 25% | 绝缘电阻、局部放电、介损 |
| ⚙️ 机械健康 | 20% | 振动、噪音、操作次数 |

### 健康等级

| 等级 | 分数范围 | 状态 | 建议操作 |
|------|----------|------|----------|
| 🟢 优秀 | 85-100 | 健康 | 常规监测 |
| 🔵 良好 | 70-84 | 正常 | 关注趋势 |
| 🟡 警告 | 50-69 | 需关注 | 计划检修 |
| 🔴 危险 | 0-49 | 异常 | 立即处理 |

## 🚀 快速开始

### 系统要求

- **操作系统**: macOS / Linux / Windows
- **Docker**: 20.x 或更高版本
- **Node.js**: 18.x 或更高版本
- **浏览器**: Chrome / Firefox / Safari (最新版)

### 一键部署

```bash
# 克隆项目
git clone https://github.com/canunited/asset-manager.git
cd canunited-asset-manager

# 启动服务
docker-compose up -d

# 访问应用
open http://localhost:3000
```
